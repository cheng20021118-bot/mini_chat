# Embedding 与相似度检索

Embedding 的目标是把文本映射到向量空间，使得语义相近的文本向量距离更近。

## 余弦相似度
余弦相似度 = (a · b) / (||a|| ||b||)。
如果向量已经做了归一化（normalize），余弦相似度等价于点积（Inner Product）。

## 为什么要 normalize
如果不归一化，向量长度会影响相似度，导致“长向量”在点积下被偏好。
因此很多检索系统会把向量 normalize 后再做内积检索。

## 检索常见问题
1. 数据太少：分数会抖动，同一个问题可能命中不稳定。
2. chunk 太大：检索命中但上下文包含太多无关内容，生成容易跑偏。
3. chunk 太小：上下文碎片化，缺关键定义。
4. top_k 设置不合理：top_k > 向量库大小可能导致无效结果，需要防御式处理。

## 简化建议
在小规模 demo 阶段，优先保证：
- 向量归一化 + IndexFlatIP（内积）
- top_k <= ntotal
- 至少保留 top1 作为兜底上下文
