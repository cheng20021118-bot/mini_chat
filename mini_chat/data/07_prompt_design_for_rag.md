# RAG Prompt 设计要点

RAG 的核心不是“把资料喂给模型就行”，而是要让模型遵守约束。

## 常用结构
1) System：设定身份与约束（只能用资料、资料不足就拒答）
2) Context：把检索到的 chunk 编号展示
3) User：用户问题

## 防止胡编的规则
- 明确写：只能使用资料中的信息，不允许引入常识或外部知识。
- 明确写：资料没有就回答“我不知道，资料中没有相关信息。”
- 可选：让模型自检（self-verification），判断回答能否从资料推出。

## 输出格式控制
不同模型输出习惯不同，有的会加“根据资料”。如果你不想要这类前缀，可以在规则里要求“直接作答，不要写根据资料”。

## 引用展示（可解释性）
把 chunk 编号成 [1][2]，并要求回答末尾给出引用编号，可以让用户复查，提升可信度。
