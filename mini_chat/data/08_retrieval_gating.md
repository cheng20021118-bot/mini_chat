# Retrieval Gating（检索门控）策略

检索门控是为了避免“检索无关资料也强行回答”。

## 典型策略
- 取 top1 的相似度 best_score
- 如果 best_score < threshold：拒答（或提示补充资料）
- 否则：把高于阈值的 chunk 作为 context

## 小数据集的特殊性
当向量库很小（例如 < 50 chunk）时：
- 分数会更抖动
- 同一个 query 可能分数波动较大

因此小库常用兜底：
- 如果过滤后 docs 为空，也至少保留 top1
这样系统行为更稳定。

## 何时调高阈值
- 数据量大、内容覆盖广
- 你想减少“牵强命中”造成的误答
